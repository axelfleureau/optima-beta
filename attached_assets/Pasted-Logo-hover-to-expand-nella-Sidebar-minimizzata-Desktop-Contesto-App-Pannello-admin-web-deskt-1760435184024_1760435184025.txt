Logo “hover-to-expand” nella Sidebar minimizzata (Desktop)
Contesto
App: Pannello admin / web (desktop).
Vista: Desktop only (≥ breakpoint lg).
Stato attuale: quando la sidebar è minimizzata, il logo in alto è statico e non ha funzione.
Obiettivo: quando la sidebar è minimizzata, il logo diventa un pulsante “re-ingrandisci” attivabile on hover (preview) e on click (toggle).
User story
Come utente desktop con sidebar minimizzata, quando passo il mouse sul logo voglio vedere un’anteprima della sidebar espansa e, cliccando, voglio ripristinare la sidebar in stato expanded per poter accedere rapidamente alle label di navigazione.
Comportamento desiderato
Hover sul logo (sidebar = minimized)
Mostra tooltip “Espandi” (o “Expand sidebar”).
Aggiunge preview: la sidebar si espande temporaneamente finché il cursore resta:
sul logo oppure
all’interno del pannello sidebar (permette di raggiungere i link).
Uscendo con il mouse dal perimetro sidebar, torna a minimized (se non c’è stato click).
Click sul logo (sidebar = minimized)
Effettua toggle permanente a expanded (salva lo stato utente).
Click sul logo (sidebar = expanded)
Facoltativo: mantiene comportamento attuale (nessuna azione) oppure toggle a minimized (confermare design decision; default: nessuna azione).
Accessibilità
Il logo, in stato minimized, è un button con aria-expanded coerente.
Focus da tastiera: Enter/Space replicano il click.
Tooltip accessibile (aria-label="Espandi sidebar" come fallback).
Animazioni
Transizione fluida 150–250 ms su width e opacity delle label.
Nessun layout shift dei contenuti principali (usare container/fixed width + CSS transitions).
Requisiti tecnici
Stato globale: sidebarState: 'expanded' | 'minimized' (Redux/Zustand/Context).
Nuovo stato volatile per hover: isHoverPreview: boolean.
Logica:
onMouseEnter sul logo → isHoverPreview = true.
onMouseLeave (logo + area sidebar) → isHoverPreview = false se sidebarState === 'minimized'.
onClick sul logo → sidebarState = 'expanded' e persist (localStorage o user prefs).
Classi suggerite (Tailwind):
Width: w-16 (minimized) → w-64 (expanded/preview).
Transizioni: transition-all duration-200 ease-out.
Label: opacity-0 pointer-events-none → opacity-100.
Z-index/hover area: includere padding invisibile (after: o wrapper) per evitare “flicker” quando si entra/uscirà tra logo e corpo sidebar.
Criteri di accettazione (QA)
 Con sidebar minimizzata, hover sul logo espande in preview entro 200 ms.
 Spostando il mouse fuori dalla sidebar, torna a minimized se non c’è stato click.
 Click sul logo in stato minimizzato porta la sidebar a expanded e resta tale anche dopo hover-out.
 Stato persistente dopo refresh (rimane expanded se così impostato).
 Il logo è percepito come button dai lettori di schermo; aria-expanded riflette lo stato.
 Navigazione da tastiera: Tab → Enter/Space funziona come click.
 Nessun salto imprevisto del layout contenuto (content area non “saltella”).
 Tooltip “Espandi” visibile su hover e focus.
 Comportamento limitato a desktop (nessun hover su touch).
 Nessun errore in console.
Out of scope
Comportamento mobile/tablet touch.
Redesign icone/brand.
Modifiche al menù voci o routing.
Note implementative (snippet indicativo – React/Tailwind)
const Sidebar = () => {
  const [sidebarState, setSidebarState] = useSidebarState(); // 'expanded' | 'minimized'
  const [hoverPreview, setHoverPreview] = useState(false);

  const isExpanded = sidebarState === 'expanded' || hoverPreview;

  return (
    <aside
      className={`group relative h-screen transition-all duration-200 ease-out 
                  ${isExpanded ? 'w-64' : 'w-16'} hidden lg:block`}
      onMouseLeave={() => {
        if (sidebarState === 'minimized') setHoverPreview(false);
      }}
    >
      <button
        className="w-full h-16 flex items-center justify-center outline-none"
        aria-label={sidebarState === 'minimized' ? 'Espandi sidebar' : 'Sidebar'}
        aria-expanded={isExpanded}
        onMouseEnter={() => {
          if (sidebarState === 'minimized') setHoverPreview(true);
        }}
        onClick={() => {
          if (sidebarState === 'minimized') {
            setSidebarState('expanded'); // + persist
            setHoverPreview(false);
          }
        }}
      >
        <Logo className="h-7 w-7" />
        {/* opzionale: mostra wordmark solo se expanded */}
        <span className={`ml-2 transition-opacity ${isExpanded ? 'opacity-100' : 'opacity-0 pointer-events-none'}`}>
          Brand
        </span>
      </button>

      <nav className="mt-2">
        {/* icone + label con opacity legata a isExpanded */}
      </nav>
    </aside>
  );
};
Metriche (facoltative)
Eventi: sidebar_expand_hover_preview, sidebar_expand_click, sidebar_state_persisted.
Successo: aumento interazioni con voci menu a sidebar minimizzata; riduzione tempo-al-primo-click menu.
